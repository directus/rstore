You have now walked through the core Nuxt workflow: collections in `app/rstore`, module-driven store setup, page queries, mutations, forms, relations, plugins, live queries, and direct cache access.

This chapter starts with the completed tutorial app already loaded in the preview, and every app file is unlocked in the editor. Treat it like a playground: change the finished code, try different store patterns, and use the guide links below when you want to push the app further.

From here, a few useful directions open up:

- [Offline](/guide/data/offline) for persisted local cache state and replayed queued mutations
- [Devtools](/guide/devtools) for embedded rstore inspection outside the built-in Nuxt DevTools tab
- [Federation](/guide/schema/federation) for routing different collections to different backends with `scopeId`
- [Nuxt + Drizzle](/plugins/nuxt-drizzle) if you want generated collections and a matching server API from a Drizzle schema
- [Yjs](/plugins/yjs) if collaboration is part of your application model

These are not separate systems bolted on afterward. They are deeper expressions of the same collection, cache, plugin, and local-first patterns you already used in the app.
